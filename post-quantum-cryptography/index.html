Step 1: The Quantum Threat to Classical Cryptography

Post-quantum cryptography (PQC) refers to cryptographic algorithms designed to resist attacks by quantum computers. Classical public-key cryptography (RSA, ECC, Diffie-Hellman) relies on the hardness of integer factorization and discrete logarithm problems. Shor's algorithm (1994) solves both in polynomial time O((log N)³) on a quantum computer, exponentially faster than the best classical algorithms (~O(e^((log N)^(1/3)})). A quantum computer with ~10⁶ logical qubits (10⁹ physical qubits with error correction) can break 2048-bit RSA in hours, estimated feasible by 2030s-2040s. This threatens all current internet security: HTTPS, TLS, VPNs, digital signatures, blockchain, and government/military communications. Symmetric cryptography (AES, SHA) is less vulnerable: Grover's algorithm provides only quadratic speedup, so AES-256 remains secure (128-bit quantum security). However, public-key cryptography must be replaced. The "harvest now, decrypt later" threat: adversaries record encrypted data today and decrypt it once quantum computers are available, compromising long-term secrets (medical records, state secrets, financial data). PQC aims to provide quantum-resistant alternatives deployable on classical computers, ensuring security in the post-quantum era.

Shor's algorithm: Factor N in O((log N)³) time
Threat timeline: 2030s-2040s (10⁹ physical qubits)
Symmetric: AES-256 → 128-bit quantum security (Grover)
Public-key: RSA, ECC, DH → broken by Shor

Step 2: NIST Post-Quantum Standardization

NIST launched a post-quantum cryptography standardization process in 2016, evaluating 82 candidate algorithms. In 2022, NIST selected 4 algorithms for standardization (finalized 2024): CRYSTALS-Kyber (lattice-based key encapsulation), CRYSTALS-Dilithium (lattice-based digital signatures), FALCON (lattice-based signatures, compact), and SPHINCS+ (hash-based signatures, stateless). Lattice-based cryptography relies on the hardness of problems like Learning With Errors (LWE) and Shortest Vector Problem (SVP), believed hard even for quantum computers. Hash-based signatures (SPHINCS+) use only hash functions (provably secure if hash is secure) but have large signatures (~50 KB). Code-based (Classic McEliece) and multivariate (Rainbow, broken 2022) were also considered. Isogeny-based (SIKE, broken 2022) was eliminated. CRYSTALS-Kyber is the primary choice for key exchange (replacing RSA/ECDH), with key sizes ~1-2 KB (vs 256 bytes for ECC). CRYSTALS-Dilithium is the primary signature scheme (replacing RSA/ECDSA), with signatures ~2-4 KB (vs 64 bytes for ECDSA). PQC algorithms are slower and have larger keys/signatures than classical, but are implementable on current hardware. Hybrid schemes (classical + PQC) provide defense-in-depth during transition.

NIST winners (2024): Kyber, Dilithium, FALCON, SPHINCS+
Lattice: LWE, SVP (hard for quantum computers)
Key sizes: Kyber ~1-2 KB (ECC: 256 bytes)
Signatures: Dilithium ~2-4 KB (ECDSA: 64 bytes)

Step 3: Lattice-Based Cryptography

Lattice-based cryptography is the leading PQC approach, underlying CRYSTALS-Kyber and Dilithium. A lattice is a discrete subgroup of ℝⁿ (e.g., integer linear combinations of basis vectors). The Learning With Errors (LWE) problem: given (A, b = As + e) where A is a random matrix, s is a secret vector, and e is small error, find s. This is believed hard even for quantum computers (no known polynomial-time quantum algorithm). CRYSTALS-Kyber (key encapsulation): Alice generates public key pk = (A, b = As + e) and secret key sk = s. Bob encapsulates a shared secret by computing c = (u = A^T r + e₁, v = b^T r + e₂ + ⌊q/2⌋·m) where m is the message (shared secret). Alice decapsulates by computing m = ⌊(v - s^T u) / (q/2)⌋. The errors e, e₁, e₂ are small, so s^T u ≈ s^T A^T r = b^T r ≈ v - ⌊q/2⌋·m, allowing recovery of m. Security relies on hardness of LWE. Kyber has three security levels: Kyber-512 (128-bit), Kyber-768 (192-bit), Kyber-1024 (256-bit). Performance: Kyber-768 key generation ~50 μs, encapsulation ~70 μs, decapsulation ~60 μs (vs RSA-2048: ~1 ms). Lattice cryptography is efficient, versatile (encryption, signatures, FHE), and quantum-resistant, making it the foundation of PQC.

LWE problem: Given (A, b = As + e), find s
Kyber: pk = (A, As + e), ciphertext = (A^T r + e₁, b^T r + e₂ + m)
Security levels: Kyber-512 (128-bit), Kyber-768 (192-bit), Kyber-1024 (256-bit)
Performance: ~50-70 μs (vs RSA: ~1 ms)

Step 4: Migration and Deployment

Migrating to PQC is a massive undertaking requiring updates to protocols (TLS 1.3, SSH, IPsec), libraries (OpenSSL, BoringSSL), and applications (browsers, VPNs, IoT devices). NIST recommends starting migration now to be quantum-safe by 2030. Challenges: larger key/signature sizes strain bandwidth and storage (especially IoT); performance overhead (though manageable); and compatibility with legacy systems. Hybrid schemes (e.g., X25519+Kyber for TLS) combine classical and PQC, providing security if either is secure. Google, Cloudflare, and AWS have deployed hybrid PQC in production. Apple announced PQC for iMessage (2024). Governments mandate PQC: US NIST standards, EU quantum-safe roadmap, China's commercial cryptography regulations. Crypto-agility (ability to switch algorithms quickly) is critical for future-proofing. PQC coexists with QKD: PQC is software-based, scalable, and works over existing networks; QKD provides unconditional security but requires specialized hardware and is limited to ~100 km. Hybrid approach: PQC for general use, QKD for high-security applications. The post-quantum transition is underway, ensuring security against future quantum computers while maintaining compatibility with classical infrastructure.

Migration: TLS 1.3, SSH, IPsec, OpenSSL, browsers
Hybrid: Classical + PQC (e.g., X25519+Kyber)
Deployment: Google, Cloudflare, AWS, Apple iMessage
Timeline: NIST recommends quantum-safe by 2030
